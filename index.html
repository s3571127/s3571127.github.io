Project Idea

A key project that I wish to see within the IT field is a cross-platform game, which operates on smartphone devices or tablets, and the two main computer operating systems, being MacOS and Windows. Depending on the success of the product, it may also be then available on console systems such as the Playstation and Xbox. The game will record and save data across all platforms, which can then be resumed on whichever platform is used next. The game feature both offline and online aspects, allowing for players to choose to complete tasks by themselves, or in a group.

Mobile gaming is a massively growing trend globally, with the industries market value expecting to reach $74.4 billion USD in 2019. This value is a 24.4% increase in market value from 2017, which boasted $59.8 billion USD. There are various games that are multi-platform compatible, however many fail to create a link between the platforms. This game will allow users to complete tasks or missions that are directly transferable between platforms. Their account will be covered over all devices, meaning that saved data is not lost by switching between phone and computer.

The product will feature two versions of the game, one for mobile operating systems and one for computer operating systems. Both games will follow the same storyline, and have similar goals in terms of progression through the game.

The main storyline and bulk of the game content will be within the computer-based game. This will be an open-world style game, with the ability to switch between solo (offline play) and multiplayer (online play) modes. The game will feature a level-based progression method, where players will be able to access better items and gear through this progression method.

Offline play will consist of a main storyline, as well as various side tasks that players can work to complete, gaining rewards for each of these. Online play will consist of various group-oriented tasks, where players must work together to complete these. It will also include an arena-style mode, where players are able to compete against others.

The data for the game will be either stored on a disc, which will then require players to install and store a fraction of the data on their hard drives, or will be completely stored on the hard drive of the user. This will include game data such as maps, character models, item models and all other essential game data. If there is no online connection, various activities will be blocked for the user until there is one present.

The phone-based game will feature various side stories and missions that are specifically tailored towards the operating system. From the completion of these missions, the player will be rewarded with content that is either able to be used across both platforms, or redeemable in the main computer game. Offline content will feature various side stories that will be on a random bi-weekly rotation, which in turn will require the game to be constantly updated to access. The bulk of the phone game will require an online connection, with various side-missions and an online Arena-based mode requiring use of the Internet.

The data for the phone-based version of the game will be stored within cloud server systems, allowing for easy and quick access for users, and less stress on servers for the company. All offline data will have to be stored within the players phone, however this will only be a small fraction of data, with a high pool of rewards randomly given on each successful completion.

Players will have the ability to play solely the computer-based game, but are heavily encouraged to use both platforms. Content will be made specifically for the phone-based version, and some rewards will be exclusively available to those who complete this content. The game will feature components in the game that will require payment, however this content will be of purely cosmetic value, or as a means to speed up the progression component of the game. All in-game rewards will be accessible to all players provided that they are using both platforms, as providing a so-called ‘pay-to-win’ option may cause players to become disheartened.

The game will need various types of software to run successfully. Software will need to be compatible across various operating systems (MacOS, iOS and Windows), and data must have the ability to be directly transferable across all of these platforms. Cloud storage must also be present to successfully run the online phone version of the game, which could also be used for the online component of the computer games. If this is not feasible, then regular servers must be established for both online versions of the game. Constant development and coding of new content, as well as a high level of creativity must be present in order to keep players interested and playing.

If the project is successful, the outcome will be having created a successful, cross-platform online multiplayer game. Depending on the level of success, it can then be reevaluated if the progression of creating console-based versions is a viable option. Another option could be the development of a completely new game, depending on how successful the cross-platform component of the project has been. The original problem of finding a game that users can enjoy everywhere will have been solved, as they can enjoy this experience either at home, or when they are out. The game will also solve the problem of having directly transferable rewards between systems, causing users to feel like they are not wasting their time focusing on either version. This development may have a large impact within the gaming scene, causing other competitors to follow suit and develop linked games that are enjoyable across all platforms.
